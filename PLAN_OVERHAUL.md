# BAZAR — план полной переработки (по ТЗ)

Составлен из 12-областного аудита (2026-07-23). Порядок — по приоритетам ТЗ:
3 главных сценария первыми, но с двумя сквозными фундаментами впереди.

## Сквозное открытие
Бэкенд уже умеет: отзывы, избранное, сохранённые поиски, уведомления, жалобы,
честный счётчик просмотров, отметки прочтения, торг. **Клиент этим не
пользуется** и рисует фейки из хеша (рейтинг ★4.8, «47 отзывов», «Проверен»,
просмотры). Первый крупный выигрыш = подключить готовое и убрать фейки.

## Фаза 1 — Дизайн-система (фундамент, низкий риск)
Токены уже есть (цвета/радиусы/тени/--sp/easing). Задача — единообразие:
- [ ] токенизировать фиолетовый AI-акцент (--ai/--ai-2 +dark), убрать raw hex, #00a06a→var(--accent) в градиентах
- [ ] типо-шкала (--fs-*/--lh-*/--weight-*), свести ~30 разнобойных размеров
- [ ] базовый .surface (фон+бордер+радиус+тень по уровням) для card/modal/auth/panel
- [ ] дисциплина бейджей: максимум 2 на карточке, единое семейство
- [ ] активировать --sp-* в частых padding/gap (или убрать мёртвые), +--radius-xs
- [ ] проверить dark + reduced-motion, НЕ трогать page-in transform (держит мобильный шит фильтров)
